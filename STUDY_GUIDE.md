# 코딩 테스트 습관화 & 실력 향상 가이드

## 핵심 마인드셋

코딩 테스트는 수학 경시대회가 아니라 운전면허 기능 시험이다.
백지에서 천재적인 로직을 발명하는 것이 아니라, **알고리즘 패턴의 수집과 체화**가 유일한 목표다.

성공 기준은 "문제를 풀었는가"가 아니라 **"오늘 로그 한 줄 남겼는가"**이다.
못 풀고 답을 본 것도 학습이다. 🔴를 모으는 것이 곧 모르는 패턴을 배우는 것이다.

---

## Phase 1: 패턴 수집기 (3~4월)

목표: Easy 유형별 패턴 익히기 + 매일 1문제 습관 정착

### 대상

NeetCode 150의 Easy 문제를 유형별로 순서대로 공략한다.
한 유형을 끝내기 전에 다른 유형으로 넘어가지 않는다.
(예: Array & Hashing의 Easy를 전부 끝내고 → Two Pointers Easy로 이동)

### 매일 워크플로우

**소요시간: 20~30분 (저녁 루틴)**

1. 문제를 읽고 **로직을 구상한다 (15분 제한)**
   - 15분 안에 접근 방식(수도코드 수준)이 떠오르지 않으면 즉시 해답을 확인한다.
   - 무의미한 시간 끌기가 의욕을 꺾는 가장 큰 원인이다. 타이머 필수.

2. 코드를 구현한다 **(15분 제한)**
   - 접근 방식은 맞았으나 구현에서 15분 이상 막히면 정답 코드를 확인한다.
   - Easy 단계에서는 구현 시간을 짧게 잡아 회전률을 높인다.

3. **결과를 기록한다 (Daily Coding Log, 30초)**

```
| 1 | LC0217 Contains Duplicate | array, hash-set | 🟢 | 8min | - |
```

   - 🟢 혼자 품 → 복습 불필요
   - 🟡 힌트/부분 참고 후 품 → 복습일에 "1개월 후 날짜" 기입
   - 🔴 답 보고 이해함 → 복습일에 "2주 후 날짜" 기입

4. **🟡/🔴일 때만 Problem Note 작성 (3분)**
   - 막힌 지점: 어디서 막혔는지 한 줄
   - 핵심 패턴: 깨달은 포인트 한 줄
   - 복습 추적 테이블에 1차 기록

### 안전장치

- **목표**: 매일 1문제
- **빨간불**: 3일 연속 공백 금지 ⚠️
- 하루 빠지는 건 괜찮다. 이틀 빠지는 것도 봐줄 수 있다. 3일 연속은 안 된다.
- 완벽주의에 짓눌려 "오늘 못했으니 내일 2개 해야지" 같은 생각은 금지. 내일도 1개면 충분하다.

## Phase 2: 패턴 심화 (5~6월)

목표: Medium 진입 + 간격반복으로 패턴 정착

### 대상

NeetCode 150의 Medium 문제를 유형별로 공략한다.
Phase 1과 동일하게 한 유형씩 집중한다.

### 변경점

- **로직 구상 시간을 20분으로 확장한다.** Medium은 Easy보다 사고 단계가 하나 더 필요하다.
- **구현 시간도 20분으로 확장한다.** 총 40분 이내.
- 그 외 기록 방식, 안전장치는 Phase 1과 동일하다.

---

## Phase 3: 패턴 판별 (7월~)

목표: 유형 태그 없이 문제를 보고 어떤 패턴인지 스스로 판별하는 능력

### 대상

NeetCode 150 범위 밖의 문제, 또는 LeetCode Random Pick을 활용한다.
유형 태그를 보지 않고 문제만 읽고 접근법을 판단하는 연습을 한다.

### 방법

1. LeetCode의 "Pick Random" 또는 NeetCode Blind 모드로 문제를 뽑는다.
2. **유형 태그를 보지 않는다.**
3. 먼저 "이 문제는 어떤 패턴인가?"를 판단하고 적는다.
4. 풀이를 시도한 뒤, 실제 태그와 비교한다.
5. 유형 판별이 맞았는지 여부를 기록에 추가한다.

### 기록 추가 항목

```
| # | 문제 | 내 판단 유형 | 실제 유형 | 판별 | 결과 | 소요시간 |
|---|------|------------|----------|------|------|----------|
| 1 | LC0xxx | sliding-window | sliding-window | ✅ | 🟡 | 25min |
```

유형 판별이 틀렸다면 (❌) → 해당 유형의 패턴 특징을 Problem Note에 다시 정리한다.

---

## 간격반복 복습 시스템

별도의 "복습 요일"을 두지 않는다.
매일 1문제 시간 안에서 신규와 복습을 자연스럽게 섞는다.

### 규칙

- 그날 복습 예정인 🔴/🟡 문제가 있으면 → **복습이 우선** (신규 문제 대신 복습)
- 복습 예정이 없으면 → 신규 문제
- 복습일에 하루 2개 이상 겹치면 → 1개만 하고 나머지는 다음날로 밀기

### 간격

```
🔴 → 2주 후 복습 → 또 🔴면 → 다시 2주 후
                    → 🟡로 올라가면 → 1개월 후 최종 확인

🟡 → 1개월 후 1회 복습 → 🟢이면 졸업
                        → 여전히 🟡면 → 다시 1개월 후

🟢 → 복습 불필요
```

### 복습 시 핵심

- 백지에서 푼다. solution.ts를 보지 않는다.
- 2주가 지나면 코드 기억은 날아가고 패턴 이해만 남는다.
- 이때 풀 수 있으면 진짜 이해한 것, 못 풀면 한 번 더 배우는 것. 어느 쪽이든 의미 있다.

python -m pytest problems/LC0217-contains-duplicate -v
